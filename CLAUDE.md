# SSWL AI Harness v2.0

태양 및 우주환경연구실(SSWL)의 연구자 업무 지원 AI 시스템.

## 프로젝트 개요

연구실에서 개발된 모델들을 **아카이빙·관리**하고, 연구자의 요청에 따라 **데이터 수집·모델 실행·결과 보고**를 수행하며, **연구 아이디어를 빠른 실험 결과**로 전환하는 하네스.

## 핵심 운용 모드

1. **연구 업무 수행**: 자연어 요청 → 계획 수립 → 승인 → 실행 → 결과 보고
2. **모델 아카이빙/활용**: 연구실 모델을 표준 형식으로 등록·관리, 누구나 활용
3. **아이디어 → 실험 결과**: 연구 아이디어 → 실험 설계 → 실행 → 결과 + 논문 초안

## 에이전트 구성

- `query-planner`: 사용자 요청 분석, 실행 계획 수립
- `data-fetcher`: 태양/우주기상 데이터 수집 (NOAA, SDO, Solar Orbiter, STEREO, VSO)
- `model-runner`: 등록된 예측/분석 모델 실행
- `task-executor`: 계획에 따른 순차/병렬 작업 실행
- `result-reporter`: 결과 정리, 사용자 보고, 후속 작업 제안
- `skill-archivist`: 모델 등록, 모델 카드 생성, 버전 관리
- `research-assistant`: 연구 아이디어 분석, 실험 설계, 논문 초안 지원

## 스킬 구성

- `research-task`: 연구 업무 오케스트레이터 (시나리오 1)
- `model-archive`: 모델 아카이빙/활용 오케스트레이터 (시나리오 2)
- `idea-to-experiment`: 아이디어→실험 오케스트레이터 (시나리오 3)
- `paper-draft`: 논문/보고서 초안 작성
- `data-acquisition`: 데이터 소스별 접근 방법 참조
- `notification`: Slack/Telegram 알림

## 작업 공간

```
_workspace/
├── plans/                  # 실행 계획
├── tasks/                  # 작업별 실행 결과
├── data/                   # 수집 데이터
├── models/                 # 모델 실행 결과
├── model_registry/         # 등록된 모델 아카이브
├── experiments/            # 실험 결과
├── papers/                 # 논문 초안
├── reports/                # 결과 보고서
└── logs/                   # 시스템 로그
```

## 사용 언어

- 사용자 대면: 한국어
- 코드/설정: 영어
- 예보문/보고서: 한국어

## 핵심 원칙

1. **사용자 승인 필수**: 모든 작업은 계획 제시 후 사용자 승인을 받아야 실행
2. **모델 표준화**: 등록된 모델은 표준 모델 카드를 갖고, 누구나 동일하게 사용 가능
3. **결과 투명성**: 성공이든 실패든, 과정과 결과를 명확히 보고
4. **연속 작업 지원**: 이전 결과를 다음 작업의 입력으로 자동 연결
