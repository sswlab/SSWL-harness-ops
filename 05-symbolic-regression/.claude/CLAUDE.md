# 05-Symbolic-Regression — SSWL Symbolic Regression 연구 하네스

태양 및 우주환경연구실(SSWL)의 **Symbolic Regression(SR)** 전문 연구 하네스.
사용자가 데이터와 과학적 질문을 제시하면, 데이터 진단 → 도구 라우팅 → 병렬 SR 실행 → 후처리/비교 → 보고까지 에이전트 팀이 협력하여 수행한다.

## 하네스의 목표

복잡한 관측/시뮬레이션 데이터로부터 **해석 가능한 닫힌 형태의 수식**을 발견하여,
1. 새로운 경험식·스케일링 법칙 제안
2. 기존 이론식의 검증·재발견·일반화
3. 관측 가능한 양들 사이의 숨은 관계 식별

을 자동화한다.

## 에이전트 팀 구성표

| 에이전트 | 역할 | 설명 |
|---|---|---|
| **sr-planner** | 진단·라우팅 | 데이터 형태/단위/노이즈 진단, 도구 선택, 실행 계획 수립 |
| **sr-executor** | 병렬 실행 | 선택된 SR 도구를 동일 조건에서 병렬 실행, 후보 수식 수집 |
| **expression-analyst** | 후처리 분석 | SymPy로 단순화·동등성 판정·복잡도 평가, Pareto front 구성 |
| **sr-reporter** | 보고 | 비교표·Figure·LaTeX 수식·해석 보고서 작성 |

## 실행 흐름

```
사용자 요청
    │
    ▼
sr-planner  →  (필요 시 사용자에게 도메인/단위 질문)
    │
    ▼  실행 계획 + 도구 라우팅
sr-executor  →  PySR / DSO / NeSymReS / KAN / AI Feynman / PySINDy / DEAP 중 선택 N개를 병렬 실행
    │
    ▼  raw 후보 수식들
expression-analyst  →  단순화, 동등성 클러스터링, Pareto (정확도 vs 복잡도)
    │
    ▼  분석된 수식 + 메트릭
sr-reporter  →  비교표, Figure, LaTeX, 해석 + 사용자 보고
```

## 활용 도구 카탈로그

| 도구 | 카테고리 | 적합한 데이터 / 상황 |
|---|---|---|
| **PySR** | GP (대표/디폴트) | 일반 (X, y) — 1순위로 항상 시도 |
| **DSO/DSR** | RL+RNN | 일반 (X, y), 토큰 제약·다른 탐색 전략 비교 필요 시 |
| **NeSymReS** | Transformer (pre-trained) | 저차원 (X, y), 빠른 1차 후보 스크리닝 |
| **pykan (KAN)** | 신경망 (Kolmogorov-Arnold) | 매끄러운 함수, 비-GP 비교군, 고차원 |
| **AI Feynman** | 물리 특화 | 물리 단위/차원 정보가 있는 경우 |
| **PySINDy** | 동역학 식별 | 시계열, ODE/PDE 형태의 시스템 |
| **DEAP** | 범용 GP 프레임워크 | 커스텀 연산자/제약 추가 fallback |
| **SymPy** | 후처리 | 모든 결과의 단순화/동등성/LaTeX (공통) |

### 도구 라우팅 정책 (sr-planner의 기본 규칙)

1. **PySR은 항상 포함** — 디폴트 워크호스. 빠지는 경우는 사용자가 명시적으로 제외할 때만.
2. **데이터가 시계열·동역학 형태**(시간축 있음, 도함수 의미가 있음) → **PySINDy 추가**.
3. **물리 단위/차원 정보가 입력에 있거나 사용자가 제공** → **AI Feynman 추가**.
4. **도메인이 모호하거나 입력에 단위 정보가 없음** → 사용자에게 질문 ("이 변수들의 물리적 의미와 단위는 무엇인가요?"). 답변이 명확한 경우에만 AI Feynman을 활성화한다.
5. **딥러닝 비교군이 필요하거나, 매끄러운 고차원 함수** → **pykan** 추가.
6. **탐색 전략 다양화** 목적이면 → **DSO** 또는 **NeSymReS** 추가 (둘 다 일반 X,y용).
7. **표준 도구로 표현 불가능한 제약**(예: 단조성, 도메인 특화 연산자)이 명시되면 → **DEAP** fallback.

라우팅 결정은 항상 **research-note.md에 근거와 함께 기록**한다.

## 필수 입력 정책

파이프라인 시작 전에 다음을 확보한다. 누락·모호 시 sr-planner가 사용자에게 질문한다.

| 항목 | 변수 | 용도 |
|---|---|---|
| 데이터 | `{데이터}` | (X, y) 배열 / 시계열 / 파일 경로 |
| 연구 질문 | `{질문}` | 무엇을 발견하고자 하는가 (예: "Φ와 v 사이 관계", "B의 시간 진화식") |
| 변수 의미·단위 | `{변수메타}` | 물리적 의미, 단위, 차원. **AI Feynman 활성화의 게이트** |
| 제약·사전지식 | `{제약}` | 사용 연산자, 양/음 부호, 단조성, 알려진 점근 거동 등 (선택) |
| 작업 경로 | `{작업경로}` | 결과물 저장 위치 |

**모호성 처리 원칙**: 목적이나 데이터 의미가 분명하지 않으면, **추측하지 말고 사용자에게 묻는다.** 특히 물리적 단위·차원 정보는 AI Feynman 활성화 여부를 좌우하므로 반드시 확보한다.

## 데이터 전달 규칙

에이전트 간 모든 데이터는 사용자가 지정한 `{작업경로}` 하위 파일로 전달한다.

| 에이전트 | 출력 파일 |
|---|---|
| sr-planner | `{작업경로}/01_diagnosis.md`, `{작업경로}/02_routing_plan.md` |
| sr-executor | `{작업경로}/code/`, `{작업경로}/results/<tool>/`, `{작업경로}/03_execution_log.md` |
| expression-analyst | `{작업경로}/04_expression_analysis.md`, `{작업경로}/results/pareto.csv` |
| sr-reporter | `{작업경로}/05_report.md`, `{작업경로}/figures/`, `{작업경로}/tables/` |
| 공통 | `{작업경로}/research-note.md` (전 과정 판단 기록 누적) |

**전달 규칙:**
1. 각 에이전트는 자신의 지정 파일에만 쓴다
2. 다른 에이전트 출력은 읽기 전용 참조
3. raw 후보 수식과 학습 로그를 포함한 모든 중간 산출물 보존
4. `research-note.md`에 모든 라우팅·해석 판단을 누적 기록

## 작업 경로 정책

- 하네스 내 `_workspace/`는 **빈 스캐폴드 템플릿**이다. 실행 결과를 여기에 저장하지 않는다.
- 이후 모든 `_workspace/` 참조는 사용자가 지정한 `{작업경로}`로 치환된다.
- harness-tmp 사용 시: 스크립트는 `python/`, 실행 로그는 날짜별 폴더에 저장한다.

## 핵심 원칙

1. **사용자 승인 필수**: sr-planner가 라우팅 계획을 제시하고, 사용자 승인 후 실행
2. **모호성은 질문으로 해소**: 변수 의미·단위·연구 목적이 모호하면 추측하지 말고 묻는다
3. **항상 다중 도구 비교**: 단일 도구의 결과만으로 결론짓지 않는다 (최소 PySR + 1개 이상)
4. **재현성**: 시드, 시간예산, 연산자 집합, 데이터 분할을 모두 명시
5. **Pareto 사고**: 정확도(R²/MSE) 만이 아니라 **복잡도(노드 수, depth)와 함께** 제시
6. **물리적 검증**: 단위 정보가 있으면 dimensional consistency 검사를 수행
7. **결과 투명성**: 실패한 도구·후보·시도도 모두 기록
8. **외부 모델 가중치**: NeSymReS 등 사전학습 모델은 첫 사용 시 다운로드 출처와 라이선스를 사용자에게 알린다
