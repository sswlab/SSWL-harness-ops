# 05-Symbolic-Regression 사용 가이드

이 하네스를 실행하기 전에 아래 항목을 준비하세요.
오케스트레이터가 시작 시 확인하며, **누락된 항목은 되물어봅니다.**

---

## 개요

데이터와 과학적 질문을 제시하면 에이전트 팀이
**데이터 진단 → 도구 라우팅 → 병렬 SR 실행 → 수식 분석 → 보고**까지 수행합니다.
관측/시뮬레이션 데이터로부터 **해석 가능한 닫힌 형태의 수식**을 자동으로 발견합니다.

---

## 필수 입력 항목

### 1. 데이터

SR의 입력이 되는 데이터를 제공합니다.

```
예: CSV 파일 경로 (/home/youn_j/data/solar_wind.csv)
예: FITS 파일
예: numpy 배열 (X, y)
```

### 2. 연구 질문

무엇을 발견하고자 하는지 알려주세요.

```
예: "자기 플럭스 Φ와 태양풍 속도 v 사이의 관계식을 찾아줘"
예: "자기장 B의 시간 진화식을 찾아줘"
예: "코로나 온도와 밀도 사이 스케일링 법칙이 있는지 확인해줘"
```

### 3. 변수 의미·단위

각 변수의 물리적 의미와 단위를 알려주세요.
이 정보는 AI Feynman 등 물리 특화 도구 활성화 여부를 결정합니다.

```
예: "X1 = 자기 플럭스 (Wb), X2 = 태양풍 속도 (km/s), y = 에너지 (erg)"
예: "단위 정보 없음" → 범용 도구만 사용
```

### 4. 제약·사전지식 (선택)

알려진 조건이 있으면 알려주세요.

```
예: "결과는 양수여야 함"
예: "단조 증가 관계가 예상됨"
예: "기존 이론식: v ∝ B^(-1/2)"
```

### 5. 작업 경로

결과물을 저장할 외부 디렉토리 경로입니다.
하네스 내부의 `_workspace/`는 빈 구조 템플릿이며, 실행 결과는 여기에 저장하지 않습니다.

```
예: /home/youn_j/experiments/sr-solar-wind
```

## 활용 도구

| 도구 | 적합한 경우 |
|---|---|
| **PySR** | 기본. 항상 포함 |
| **PySINDy** | 시계열, ODE/PDE 형태 |
| **AI Feynman** | 물리 단위/차원 정보 있을 때 |
| **pykan (KAN)** | 매끄러운 함수, 고차원 |
| **DSO/DSR** | 탐색 전략 다양화 |
| **NeSymReS** | 저차원, 빠른 스크리닝 |
| **DEAP** | 커스텀 연산자/제약 |

## 에이전트 구성

| 에이전트 | 역할 |
|---|---|
| sr-planner | 데이터 진단, 도구 선택, 실행 계획 수립 |
| sr-executor | 선택된 SR 도구를 병렬 실행, 후보 수식 수집 |
| expression-analyst | SymPy로 단순화, 동등성 판정, Pareto front 구성 |
| sr-reporter | 비교표, Figure, LaTeX 수식, 해석 보고서 작성 |

## 실행 흐름

```
데이터 + 연구 질문
    │
    ▼
sr-planner (진단 + 도구 라우팅)
    │
    ▼
sr-executor (PySR + 선택 도구 N개 병렬 실행)
    │
    ▼
expression-analyst (단순화, 클러스터링, Pareto)
    │
    ▼
sr-reporter (비교표, Figure, 해석 보고서)
```
