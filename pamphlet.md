# SSWL AI Harness v2.0

### 태양 및 우주환경연구실을 위한 연구 지원 AI 시스템

**모델 아카이빙 · 연구 업무 수행 · 아이디어→실험 결과 · 논문 초안 작성**

---

*"연구실에서 개발된 모델을 누구나 활용하고, 아이디어를 빠르게 검증합니다."*

</div>

---

## 이 시스템은 왜 필요한가요?

태양 및 우주환경 연구실에서 흔히 겪는 문제들:

| 기존 방식 | 문제점 |
|---|---|
| 연구실 모델이 개발자 PC에만 존재 | 개발자가 부재하면 아무도 실행하지 못함 |
| 모델 실행 방법이 구전(口傳)으로 전수 | 새 연구자가 합류하면 재교육 필요 |
| 데이터 수집을 사이트별로 수동 수행 | 연구 시간의 상당 부분이 데이터 수집에 소요 |
| 교수님 아이디어가 실험까지 오래 걸림 | 아이디어 → 데이터 수집 → 실행 → 분석 과정이 수일~수주 |
| 모델 버전 관리 부재 | 어떤 버전으로 어떤 결과를 냈는지 추적 불가 |

> **SSWL AI 하네스 v2.0**은 위 문제들을 해결합니다.
> 연구실 모델을 **표준화·아카이빙**하고, 누구나 **자연어로 실행**하며,
> 아이디어를 **빠르게 실험 결과**로 전환합니다.

## 세 가지 핵심 기능

### 기능 1. 연구 업무 수행 — "말하면 해줍니다"

연구자가 **자연어**로 작업을 요청하면, AI가 계획을 세우고 승인 후 자동으로 실행합니다.

#### 요청 예시

> *"최근 1달 HMI 데이터로 synoptic map 만들어줘"*
>
> *"AIA 193 이미지에서 코로나홀 찾아줘"*
>
> *"이걸로 PFSS 시뮬레이션 돌려줘"*

#### 처리 흐름

```
  연구자 요청
      │
      ▼
  ┌─────────────┐     ┌──────────────┐
  │  계획 수립   │────▶│  계획서 제시  │
  └─────────────┘     └──────┬───────┘
                             │
                       연구자 승인 ✓
                             │
                             ▼
                    ┌─────────────────┐
                    │  자동 실행       │
                    │  (데이터 수집 →  │
                    │   모델 실행 →    │
                    │   후처리)        │
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  결과 보고서     │──▶ 후속 작업 제안
                    └─────────────────┘    (반복 가능)
```

> **핵심 원칙:** AI가 임의로 실행하지 않습니다.
> 반드시 연구자의 **승인**을 거친 후에만 작업이 시작됩니다.

---

### 기능 2. 모델 아카이빙 & 활용 — "개발하지 않았어도 사용합니다"

연구실에서 개발된 모델을 **표준 형식으로 등록**하면, 개발에 참여하지 않은 연구자도
**동일하게 사용**할 수 있습니다.

#### 모델 등록

```
개발자: "코로나홀 탐지 모델 등록해줘. 코드는 여기에."
    │
    ▼
AI가 코드를 분석하여 모델 카드(사용 설명서)를 자동 생성
    │
    ▼
모델 레지스트리에 등록 완료!
```

#### 모델 활용 (개발 미참여 연구자)

```
연구자B: "코로나홀 탐지 모델로 어제 AIA 193 이미지 분석해줘"
    │
    ▼
AI가 모델 카드를 참조하여:
  1. 필요 데이터 자동 수집 (JSOC에서 AIA 193)
  2. 모델 실행
  3. 결과 시각화 및 보고
```

#### 모델 목록 조회

> *"우리 연구실에 어떤 모델들이 있어?"*

```
┌─────────────────────────────────────────────────────┐
│  # │ 모델명              │ 용도         │ 개발자    │
│────┼─────────────────────┼──────────────┼──────────│
│  1 │ coronal_hole_detect │ 코로나홀 탐지 │ 김연구   │
│  2 │ dem_model           │ DEM 분석     │ 이연구   │
│  3 │ synoptic_map        │ Synoptic map │ (내장)   │
│  4 │ pfss_sim            │ PFSS 시뮬    │ (내장)   │
└─────────────────────────────────────────────────────┘
```

### 기능 3. 아이디어 → 실험 결과 — "말하면 실험합니다"

교수님이나 연구자가 **아이디어를 말하면**, AI가 **실험을 설계하고 실행**하여
**결과를 보여줍니다**. 논문 초안까지 작성해 줍니다.

#### 사용 예시

```
교수님: "Solar Orbiter DEM에 Magnetogram을 추가하면 성능이 좋아질까?"

  AI → 실험 설계:
    ├─ Baseline: EUI만으로 DEM
    ├─ Experiment: EUI + PHI(magnetogram)로 DEM
    └─ Reference: AIA DEM (비교 기준)

  AI → 실험 실행:
    ├─ 데이터 자동 수집 (EUI, PHI, AIA)
    ├─ 3가지 DEM 생성
    └─ 비교 분석

  AI → 결과 보고:
    "Magnetogram 추가 시 활동영역에서 12% 개선.
     조용한 영역은 유의미한 차이 없음."
    + 비교표, 그래프 제공

교수님: "이걸로 짧은 논문 초안 써줘"

  AI → 논문 초안 (Introduction ~ Discussion) 작성
```

## 접근 가능한 데이터 소스

전 세계 주요 태양·우주기상 관측 데이터를 **하나의 인터페이스**로 접근합니다.

| 데이터 소스 | 제공 데이터 | 갱신 주기 |
|---|---|---|
| **NOAA SWPC** | 태양풍 플라즈마, IMF, 플레어, Kp, Dst | 1~5분 |
| **SDO (AIA)** | 극자외선 태양 영상 (7개 파장) | 12초 |
| **SDO (HMI)** | 자기장 관측, 시놉틱 맵 | 45초 |
| **Solar Orbiter** | EUI 태양 영상, PHI 자기장 | 미션별 |
| **STEREO** | 코로나그래프, EUVI, 태양풍 입자 | 미션별 |
| **VSO** | SOHO, GOES, 지상 관측소 통합 검색 | 소스별 |

## 기대 효과

### 연구자 개인에게

| 항목 | 기존 | 도입 후 |
|---|---|---|
| 모델 사용 | 개발자에게 의존, 구전 교육 | 자연어로 누구나 즉시 실행 |
| 데이터 수집 | 여러 사이트 수동 다운로드 | AI가 자동 수집·검증 |
| 아이디어 검증 | 데이터 수집~분석까지 수일 | 아이디어 제시 후 당일 결과 확인 |
| 모델 실행 | 명령어 직접 입력, 환경 세팅 | 자연어 요청으로 자동 실행 |
| 논문 초안 | 백지부터 직접 작성 | 실험 결과 기반 골격 자동 생성 |

### 연구실 차원에서

- **모델 자산화**: 개발된 모델이 개인 PC에 묻히지 않고, 연구실 자산으로 관리
- **지식 전수**: 신규 연구자도 등록된 모델을 즉시 활용 가능
- **연구 가속**: 아이디어에서 초기 결과까지의 시간을 대폭 단축
- **재현성 확보**: 모델 버전 관리로 어떤 버전의 어떤 결과인지 추적 가능
- **접근성 향상**: 코딩 없이 자연어로 데이터 처리·모델 실행 가능

## 시스템 안전 장치

| 안전 장치 | 설명 |
|---|---|
| **사용자 승인 필수** | 모든 작업은 반드시 연구자 승인 후 실행 |
| **모델 버전 관리** | 업데이트 시 이전 버전 자동 보존 |
| **장애 자동 복구** | 데이터 소스 장애 시 대체 소스 자동 전환 |
| **단계별 오류 격리** | 한 단계 실패가 전체 시스템에 영향을 주지 않음 |
| **전 과정 로깅** | 모든 판단과 실행 기록이 남아 추적·감사 가능 |
| **결과 투명성** | 성공이든 실패든, 과정과 결과를 명확히 보고 |
