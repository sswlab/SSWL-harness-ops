# 06-Paper-Editor 사용 가이드

이 하네스를 실행하기 전에 아래 항목을 준비하세요.
오케스트레이터가 시작 시 확인하며, **누락된 항목은 되물어봅니다.**

---

## 개요

기존 논문(.tex, .pdf, .md)을 입력하면,
**문법 교정 / 팩트체크 / 모의 피어리뷰 / 구조 개선 제안** 중 원하는 작업을 선택하여 수행합니다.
원본 파일은 수정하지 않으며, 교정본과 리포트를 별도 파일로 생성합니다.

> **범위**: 기존 논문의 **품질 개선**(교정/리뷰)을 담당합니다.
> 연구 결과로부터 새 논문을 작성하는 것은 이 하네스의 범위 밖입니다.

---

## 필수 입력 항목

### 1. 논문 파일

편집 대상 논문 파일의 경로를 제공합니다.

```
예: /home/youn_j/papers/my_paper.tex
예: /home/youn_j/papers/draft.pdf
예: /home/youn_j/papers/manuscript.md
```

### 2. 작업 모드

어떤 작업을 수행할지 선택합니다. 복수 선택 가능합니다.

| 모드 | 설명 |
|---|---|
| **문법·표현 교정** | 영문 문법, 학술 표현, 용어 일관성, 오타 교정 |
| **팩트체크** | 모든 문장을 한 문장씩 검증, 출처 없는 주장에 레퍼런스 추천 |
| **모의 리뷰** | 리뷰어가 한 줄씩 팩트체크 + 방법론/해석 평가 (1회) |
| **구조 개선 제안** | 섹션 구성, 논리 흐름, Figure/Table 배치 개선안 |

요청에서 모드를 특정한 경우("리뷰해줘", "문법 수정해줘") 메뉴 없이 바로 진입합니다.

### 3. 보조 자료 (선택)

팩트체크 시 대조할 자료가 있으면 제공합니다.

```
예: 연구 코드, 데이터 파일, Figure 원본
```

### 4. 작업 경로

결과물을 저장할 디렉토리입니다. 입력 논문과 동일 디렉토리를 사용해도 됩니다.

```
예: /home/youn_j/papers/my_paper/
```

---

## 실행 예시

### 모드를 명시하는 경우

```
이 논문 문법 교정 + 모의 리뷰 해줘: /home/youn_j/papers/draft.tex
작업 경로: /home/youn_j/papers/draft-review/
```

### 간단하게 요청하는 경우

```
이 논문 리뷰해줘: /home/youn_j/papers/my_paper.pdf
```

→ 모의 리뷰 모드로 바로 진입, 작업 경로를 되물음

### 복수 모드 선택

```
이 논문 전체적으로 점검해줘: /home/youn_j/papers/draft.tex
```

→ 4개 모드 선택 메뉴 제시 (복수 선택 가능)

## 에이전트 구성

| 에이전트 | 역할 | 투입 모드 |
|---|---|---|
| grammar-editor | 문법·표현·일관성 교정 | 문법 교정 |
| fact-checker | 수치·주장을 코드/데이터/Figure와 대조 검증 | 팩트체크 |
| structure-advisor | 섹션 구성, 논리 흐름, Figure/Table 배치 분석 | 구조 개선 |
| reviewer | 한 줄씩 팩트체크 + 방법론/해석 평가 | 모의 리뷰 |
| reference-finder | NASA ADS / arXiv에서 참고문헌 검색·추천 | 팩트체크, 모의 리뷰 |

## 작업 공간 구조

```
{작업경로}/
├── grammar/
│   ├── edited_draft.md       # 교정된 원고
│   └── grammar_report.md     # 교정 리포트
├── factcheck/
│   └── factcheck_report.md   # 팩트체크 결과
├── structure/
│   └── structure_report.md   # 구조 개선 제안
├── review/
│   └── review_report.md      # 모의 리뷰 리포트
├── references/
│   └── reference_report.md   # 추천 참고문헌
└── editorial-log.md          # 전 과정 진행 기록
```
