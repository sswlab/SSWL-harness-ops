# DEM-Pipeline — DEM 계산 파이프라인 총괄

## 개요

사용자가 6개 파장 이미지와 응답 함수를 제시하면, 데이터 검증→DEM 역산→시각화→품질 검토의
전 과정을 자동으로 조율한다. DEMreg / SITES / 양쪽 비교 모드를 지원한다.

---

## 필수 입력 확인 및 검증

**파이프라인 시작 전, 아래 항목을 반드시 사용자에게 질문하여 확보한다.**
**절대 자동 감지/추정하지 않는다. 모든 항목을 명시적으로 확인받는다.**

누락 항목이 하나라도 있으면 아래 질문 템플릿을 사용하여 되물어서 확보한다.

```
DEM 계산에 필요한 정보를 확인합니다:

1. 관측 날짜:
2. 기기/파장: AIA (94,131,171,193,211,335) / EUI-FSI (94,131,174,193,211,335) / 직접 입력
3. 데이터 위치: (디렉토리 경로)
4. 데이터 포맷: .npy / .npz / .fits
5. 응답 함수: 번들 AIA / 번들 EUI / 직접 제공 (경로)
6. 역산 방법: DEMreg / SITES / 양쪽 비교
7. 온도 범위: (기본: log T = 5.6–7.0)
8. 작업 경로: (결과 저장 디렉토리)

빈 항목이 있으면 알려주세요.
```

| # | 항목 | 변수 | 누락 시 질문 |
|---|---|---|---|
| 1 | **관측 날짜** | `{날짜}` | "관측 날짜를 알려주세요. (예: 2024-03-20)" |
| 2 | **기기/파장** | `{기기}`, `{파장}` | "어떤 기기의 데이터인가요? AIA (171Å) / EUI-FSI (174Å) / 직접 입력" |
| 3 | **데이터 위치** | `{데이터경로}` | "6개 파장 이미지 파일이 있는 디렉토리 경로를 알려주세요." |
| 4 | **데이터 포맷** | `{포맷}` | "파일 포맷이 뭔가요? (.npy / .npz / .fits)" |
| 5 | **응답 함수** | `{응답함수}` | "응답 함수를 선택해주세요: 번들 AIA / 번들 EUI / 직접 제공 (경로)" |
| 6 | **역산 방법** | `{방법}` | 아래 방법 선택지를 제시 |
| 7 | **온도 범위** | `{온도범위}` | "DEM 온도 범위를 지정해주세요. (기본: log T = 5.6–7.0)" |
| 8 | **작업 경로** | `{작업경로}` | "결과물을 저장할 작업 경로를 알려주세요." |

### 기기/파장 선택지

```
어떤 기기의 데이터인가요?

[1] SDO/AIA
    - 파장: 94, 131, 171, 193, 211, 335 Å
    - 응답 함수: 번들 AIA (resp_concat_aia)

[2] Solar Orbiter/EUI (FSI)
    - 파장: 94, 131, 174, 193, 211, 335 Å (174가 171 대체)
    - 응답 함수: 번들 EUI (resp_concat_FSI_factor0.7)

[3] 직접 입력
    - 파장 목록과 응답 함수를 직접 지정
```

### 역산 방법 선택지

```
역산 방법을 선택해주세요:

[1] DEMreg (정규화 기반)
    - GSVD + Tikhonov 정규화 (Hannah & Kontar 2013)
    - 정밀한 에러 추정 (수직 + 수평)
    - 적합: 소규모 영역의 정밀 분석

[2] SITES (반복적 가중 역산)
    - Grid-SITES 비닝 가속 (Morgan 2019)
    - 대규모 이미지 효율적 처리
    - 적합: 전면 디스크, 대규모 맵

[3] 양쪽 비교
    - DEMreg와 SITES 모두 실행
    - 결과 차이 비교 보고서 생성
    - 적합: 방법론 비교, 결과 교차 검증
```

## 작업 경로 설정

1. 사용자가 출력 경로를 명시한 경우 → 해당 경로를 `{작업경로}`로 사용
2. 경로가 확정되면 하위 디렉토리 생성:
   ```
   {작업경로}/
   ├── data/           # 입력: 6채널 이미지 (.npy/.npz/.fits)
   │                   #   파일명에 파장 포함: aia_94.npy, data_131.fits 등
   │                   #   또는 단일 .npz (키: '94','131',... 또는 'dn_cube')
   │                   #   에러 파일 선택: err_94.npy 등 (없으면 자동 추정)
   ├── response/       # 입력: 응답 함수
   │                   #   tresp.npy + tresp_logt.npy
   │                   #   또는 .npz (키: 'tresp', 'tresp_logt')
   ├── results/        # 출력: DEM, 에러, chi-sq, 합성 DN
   ├── figures/        # 출력: DEM 맵, EM loci, chi-sq 맵
   ├── reports/        # 출력: 품질 보고서
   └── logs/           # 출력: dem-note.md
   ```

### 데이터 로드 (core.loader)

data-validator가 사용자 확인된 파라미터로 `core.loader.load_data()`를 호출한다.
**파장, 기기, 포맷은 사용자가 확인한 값을 명시적으로 전달한다. 자동 추정 금지.**

```python
from core.loader import load_data, load_response

# 사용자가 확인한 파장을 명시적으로 전달
data = load_data('{데이터경로}', wavelengths={파장})  # 예: [94,131,174,193,211,335]
resp = load_response('{응답함수경로}', wavelengths={파장})
```

## 파이프라인 흐름도

```
┌───────────────────────────────────────────────────────┐
│  사용자: 6채널 데이터 + 응답 함수 + 역산 방법 선택      │
└──────────────────────────┬────────────────────────────┘
                           │
                           ▼
              ┌────────────────────────────┐
              │  Phase 1: 데이터 검증       │
              │  data-validator            │
              │  → data/, response/        │
              │  (로드, 검증, 전처리, 에러)  │
              └───────────┬────────────────┘
                          │
                     PASS / FAIL
                     │         │
                   PASS      FAIL → 사용자에게 문제 보고
                     │
                     ▼
              ┌────────────────────────────┐
              │  Phase 2: DEM 역산         │
              │  dem-calculator            │
              │  → results/                │
              │  (DEMreg / SITES / 양쪽)   │
              └───────────┬────────────────┘
                          │
                          ▼
              ┌────────────────────────────┐
              │  Phase 3: 시각화           │
              │  result-visualizer         │
              │  → figures/                │
              │  (DEM 맵, EM loci, chi-sq) │
              └───────────┬────────────────┘
                          │
                          ▼
              ┌────────────────────────────┐
              │  Phase 4: 품질 검토         │
              │  quality-reviewer          │
              │  → reports/                │
              └───────────┬────────────────┘
                          │
                 ┌────────┴────────┐
                 │                 │
              REVISE             PASS
                 │                 │
                 ▼                 ▼
          dem-calculator      ┌────────────────────────────┐
          (파라미터 조정,      │  최종 결과 사용자에게 전달    │
           최대 2회)          │  + 품질 보고서 + 피규어      │
                              └────────────────────────────┘
```

### 양쪽 비교 모드

```
Phase 2에서:
  ├── dem-calculator (DEMreg) → results_demreg/
  └── dem-calculator (SITES)  → results_sites/

Phase 3에서:
  └── result-visualizer → 비교 플롯 생성
      (나란히 배치, 차이맵, 상관 플롯)

Phase 4에서:
  └── quality-reviewer → 비교 보고서
      (어떤 방법이 이 데이터에 더 적합한지 판단)
```

## 루프백 조건과 최대 횟수

| 루프백 유형 | 트리거 | 최대 횟수 | 초과 시 |
|---|---|---|---|
| **역산 REVISE** | quality-reviewer가 REVISE 판정 | 2회 | 사용자에게 현 상태 보고, 수동 파라미터 조정 안내 |
| **데이터 FAIL** | data-validator가 FAIL 판정 | 0회 | 사용자에게 즉시 문제 보고 |

---

## 에러 핸들링 테이블

| Phase | 에러 유형 | 심각도 | 대응 |
|---|---|---|---|
| 준비 | 데이터 파일 없음 | 높음 | 경로 재확인 요청 |
| 1 | 채널 수 부족 | 높음 | 사용자에게 확인 |
| 1 | 응답 함수 없음 | 중간 | aiapy 자동 생성 시도 |
| 2 | SOTA 코드 import 실패 | 높음 | 경로 확인, 의존성 설치 안내 |
| 2 | 메모리 부족 | 중간 | 타일링으로 분할 재시도 |
| 3 | 시각화 실패 | 낮음 | matplotlib backend 확인 |
| 4 | REVISE 2회 초과 | 중간 | 사용자 에스컬레이션 |
