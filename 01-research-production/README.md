# 01-Research-Production 사용 가이드

이 하네스를 실행하기 전에 아래 항목을 준비하세요.
오케스트레이터가 시작 시 확인하며, **누락된 항목은 되물어봅니다.**

---

## 필수 입력 항목

### 1. 연구 주제

연구하고자 하는 주제를 자연어로 제시합니다.

```
예: "GOES XRS 데이터를 이용한 태양 플레어 발생 예측"
예: "SDO/AIA 다중 파장 코로나 온도 분포 분석"
```

### 2. 연구 목적

**왜** 이 연구를 하는지, 최종적으로 달성하고자 하는 것이 무엇인지 1~2문장으로 명시합니다.
이 목적은 파이프라인 전체에 걸쳐 모든 에이전트의 판단 기준이 됩니다.

```
예: "STIX 단독 관측만으로 GOES 등급을 추정할 수 있게 하여, Solar Orbiter의 독립적 플레어 분류 역량을 확보한다."
예: "기존 DEM 방법 대비 계산 속도를 10배 향상시키면서 정확도를 유지하는 경량 모델을 개발한다."
예: "태양 플레어 예측 분야의 최근 5년 연구 동향을 정리하고, 미해결 과제를 식별한다."
```

연구 목적이 명확할수록:
- literature-reviewer가 관련성 높은 논문을 선별합니다
- research-designer가 목적에 맞는 실험을 설계합니다
- reviewer가 결과가 목적을 달성했는지 판정합니다
- paper-writer가 논문의 contribution을 정확히 서술합니다

### 3. 연구 모드

연구의 범위와 깊이를 사전에 선택합니다.

| 모드 | 설명 | 적합한 경우 |
|---|---|---|
| **탐색형 (Survey)** | 넓은 범위 문헌조사 중심. 실험 최소/생략. 리뷰 페이퍼 스타일 산출물. | 분야 동향 파악, 리뷰 논문 작성, 새 프로젝트 착수 전 조사 |
| **심층형 (Deep Dive)** | 특정 주제에 집중. 가설→실험→검증 전체 수행. 오리지널 리서치 산출물. | 구체적 가설 검증, 새 모델/방법 개발, 저널 투고용 연구 |
| **전체 (Full Pipeline)** | 넓은 문헌조사 + 깊은 실험 모두 수행. | 새 분야 진입 + 첫 논문, 종합적 연구 |

#### 모드별 Phase 비중

```
              Phase 1     Phase 2     Phase 3     Phase 4     Phase 5
              문헌조사    연구설계    실험실행    품질검토    논문작성
탐색형        ████████    ██          ░░          ████        ██████
심층형        ███         ██████      ████████    ██████      ██████
전체          ██████      ██████      ████████    ██████      ██████

█ = 핵심 작업    ░ = 최소/생략
```

### 4. 작업 경로

결과물을 저장할 외부 디렉토리 경로입니다.
하네스 내부의 `_workspace/`는 빈 구조 템플릿이며, 실행 결과는 여기에 저장하지 않습니다.

```
예: /home/youn_j/projects/flare-prediction/_workspace
예: /home/youn_j/Harness-first-study/_workspace-V5
```

## 실행 예시

### 모든 항목을 한번에 제시하는 경우

```
GOES XRS 시계열에서 M급 이상 플레어를 24시간 전에 예측하는 ML 모델을 연구해줘.

목적: 기존 임계값 기반 예측 대비 ML의 우위를 정량적으로 입증하여 ApJ 투고용 논문을 생산한다.
모드: 심층형
경로: /home/youn_j/projects/flare-prediction/_workspace
```

### 주제만 제시하는 경우

```
코로나 온도 분포 분석 연구해줘.
```

→ 오케스트레이터가 순차적으로 되물음:
1. "연구 목적을 알려주세요."
2. "연구 모드를 선택해주세요. [1] 탐색형 [2] 심층형 [3] 전체"
3. "결과물을 저장할 작업 경로를 알려주세요."
