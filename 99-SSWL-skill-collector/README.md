# 99-SSWL-Skill-Collector 사용 가이드

이 하네스를 실행하기 전에 아래 항목을 준비하세요.
오케스트레이터가 시작 시 확인하며, **누락된 항목은 되물어봅니다.**

---

## 코드 넣는 곳

변환할 코드는 **`_workspace/inbox/`** 디렉토리에 넣으시면 됩니다.
정리 안 된 상태 그대로 넣으셔도 됩니다 — code-archaeologist 에이전트가 읽고 분석합니다.

```
99-SSWL-skill-collector/_workspace/inbox/   ← 여기에 코드 파일 복사
```

또는 외부 경로에 있는 코드를 직접 지정할 수도 있습니다.

## 원격 서버에서 수집하기 (선택)

다른 서버에 있는 코드를 수집하여 변환할 수도 있습니다.
이 경우 코드를 직접 inbox/에 넣지 않아도 됩니다 — **remote-collector가 자동으로 수집**합니다.

필요한 추가 정보:
- SSH 접속 정보: `user@hostname` (예: `youn_j@163.180.171.99`)
- 스캔 대상 경로: 원격 서버의 코드 디렉토리 (예: `/userhome/youn_j/`)

수집 과정:
1. 원격 서버를 스캔하여 파일 목록을 보여드립니다
2. 버전이 여러 개인 코드는 **최신/최완전 버전을 자동 선별**합니다
3. 선별 결과를 확인받은 후 전송합니다
4. 전송된 코드는 **범주별로 inbox/에 정리**됩니다

> SSH 키 기반 인증이 설정되어 있어야 합니다. `ssh-copy-id user@hostname`으로 사전 설정하세요.

## 필수 입력 항목

### 1. 코드 위치

변환할 코드가 있는 디렉토리 경로를 알려주세요.
`_workspace/inbox/`에 넣었으면 별도 지정 불필요합니다.

```
예: _workspace/inbox/  (기본)
예: /home/youn_j/lab-codes/
예: /home/youn_j/old-scripts/stix/
```

### 2. 변환 목적

**왜** 이 코드를 스킬로 만드는지 알려주세요.
목적에 따라 분류 기준과 패키징 수준이 달라집니다.

```
예: "연구실 신입생이 바로 사용할 수 있는 공용 도구로 만들기"
예: "기존 데이터 파이프라인을 보강하는 유틸리티로 만들기"
예: "자주 쓰는 코드를 스킬로 등록해서 매번 다시 짜지 않기"
```

### 3. 변환 모드

| 모드 | 설명 | 적합한 경우 |
|---|---|---|
| **전체 변환 (Full Scan)** | 모든 코드를 분석·분류·변환 | 처음으로 코드를 정리할 때 |
| **추가 수집 (Incremental)** | 기존 분류에 새 코드 추가 | 1차 정리 이후 새 코드가 생겼을 때 |
| **단일 변환 (Targeted)** | 특정 파일만 스킬로 변환 | 변환 대상이 명확할 때 |
| **원격 수집 + 전체 변환** | 원격 서버에서 수집 후 전체 변환 | 다른 서버에 코드가 흩어져 있을 때 |

### 4. 작업 경로

중간 결과물(인벤토리, 분류, 모듈, 스킬 초안)을 저장할 디렉토리입니다.
하네스 내부의 `_workspace/`는 빈 구조 템플릿이며, 실행 결과는 여기에 저장하지 않습니다.

```
예: /home/youn_j/skill-collector-workspace
```

## 실행 예시

### 모든 항목을 한번에 제시하는 경우

```
연구실 코드를 스킬로 정리해줘.

코드 위치: /home/youn_j/lab-codes/
목적: 연구실 공용 도구로 만들어서 신입생도 사용할 수 있게
모드: 전체 변환
경로: /home/youn_j/skill-collector-workspace
```

### 코드 위치만 제시하는 경우

```
/home/youn_j/lab-codes/에 있는 코드 정리해줘.
```

→ 오케스트레이터가 순차적으로 되물음:
1. "변환 목적을 알려주세요."
2. "변환 모드를 선택해주세요. [1] 전체 변환 [2] 추가 수집 [3] 단일 변환"
3. "작업 경로를 알려주세요."

### 특정 파일만 변환하는 경우

```
이 파일만 스킬로 만들어줘: /home/youn_j/lab-codes/dem_calculator.py
```

→ 단일 변환 모드로 자동 진행

### 원격 서버 코드를 수집하여 변환하는 경우

```
99서버에 있는 연구 코드를 스킬로 정리해줘.

서버: youn_j@163.180.171.99:/userhome/youn_j/
목적: 흩어진 코드를 재사용 가능한 도구로 정리
모드: 원격 수집 + 전체 변환
경로: /home/youn_j/skill-collector-workspace
```

→ remote-collector가 서버 스캔 → 버전 선별 → 승인 → 전송 → 나머지 파이프라인 자동 진행

## 파이프라인 흐름

```
[원격 수집] → 코드 덤프 → 분석 → 분류 → [승인] → 리팩터링 → 스킬 패키징 → 검증 → 배포
```

- **원격 수집 후 승인 단계**: 수집할 파일 목록을 먼저 확인받습니다 (원격 모드 전용).

- **분류 후 승인 단계**가 있습니다: 분류 결과를 확인하고, 수정할 부분이 있으면 알려주세요.
- **원본 코드는 절대 수정하지 않습니다**: 모든 작업은 사본으로 수행합니다.
- **과정이 기록됩니다**: `{작업경로}/logs/collector-note.md`에 모든 판단 과정이 누적됩니다.

## 작업 공간 구조

```
{작업경로}/
├── collection/     # Phase 0: 원격 수집 매니페스트 (원격 모드 시)
├── inventory/      # Phase 1: 코드 분석 결과 (버전 계보, 노트북 분석 포함)
├── clusters/       # Phase 2: 기능별 분류 결과
├── modules/        # Phase 3: 리팩터링된 모듈
├── skills/         # Phase 4: 패키징된 스킬 초안
├── reports/        # Phase 5: 검증 보고서
└── logs/           # 전 과정 기록
```
