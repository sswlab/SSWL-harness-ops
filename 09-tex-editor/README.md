# 09-TeX-Editor 사용 가이드

LaTeX 원고(`.tex`)를 입력하면 **컴파일 → 시각·문법 검토 → 수정된 `.tex` 출력**까지 수행하는 AI LaTeX 편집 하네스.

GPT Prism과 유사한 역할을 로컬에서 에이전트 팀으로 수행한다.

---

## 개요

- **입력**: 루트 `.tex` 파일 + 보조 파일(그림, `.bib`, 클래스/스타일 파일)이 있는 디렉토리
- **출력**: 수정된 `.tex` 파일 + 변경 리포트

**검토 항목:**
- LaTeX 컴파일 에러·경고 (undefined refs, missing figures 등)
- **시각 문제**: 표 잘림, 그림 마진 벗어남, 캡션 분리, 페이지 경계 이상, 수식 줄바꿈 깨짐
- **문법·오타**: 영문 문법 오류 및 오탈자 (내용은 건드리지 않음)

---

## 필수 입력 항목

### 1. 루트 `.tex` 파일 경로

```
예: /home/youn_j/papers/my_paper/main.tex
```

### 2. 작업 경로

컴파일 중간물·PNG·리포트·수정본을 저장할 디렉토리.

```
예: /home/youn_j/papers/my_paper/_tex-editor/
```

입력 `.tex` 디렉토리와 동일해도 무방하나, 별도 권장.

---

## 실행 예시

```
이 tex 파일 편집해줘: /home/youn_j/papers/my_paper/main.tex
작업경로: /home/youn_j/papers/my_paper/_tex-editor/
```

---

## 에이전트 구성

| 에이전트 | 역할 |
|---|---|
| **latex-compiler** | `latexmk`로 컴파일, 로그 파싱(에러·경고), `pdftoppm`으로 페이지 PNG 렌더링 |
| **visual-inspector** | 렌더된 PNG를 직접 보고 시각 문제 탐지 (표 잘림, 그림 마진 벗어남, 캡션 분리, 페이지 경계 이상 등) |
| **grammar-checker** | 영문 문법·오타만 검토. 과학적 내용은 건드리지 않음 |
| **tex-editor** | 3개 리포트를 받아 수정본 `.tex` 생성 |

---

## 작업 공간 구조

```
{작업경로}/
├── source/           # 원본 .tex 및 보조파일 복사본
├── build/            # 컴파일 산출물 (.aux, .log, .pdf)
├── pages/            # page-001.png ...
├── reports/
│   ├── compile.md    # 컴파일 로그 분석
│   ├── visual.md     # 시각 문제 리포트
│   └── grammar.md    # 문법·오타 리포트
├── revised.tex       # 최종 수정본
└── editorial-log.md  # 전 과정 진행 기록
```

---

## 핵심 원칙

1. **원본 비파괴**: 입력 `.tex` 디렉토리를 수정하지 않는다. `source/`에 복사 후 작업
2. **1회 실행 후 사용자 보고**: 자동 재컴파일 루프 없음. 수정본 생성 후 사용자가 승인하면 재실행
3. **시각 문제는 실제로 본다**: 로그 기반 경고(overfull hbox)뿐 아니라 PNG를 직접 관찰하여 판단
4. **문법 교정은 보수적**: 명백한 문법 오류와 오타만 수정. 저자 문체는 존중
5. **컴파일 에러는 최우선**: 에러가 있으면 수정본의 첫 번째 책무는 컴파일이 되도록 만드는 것
