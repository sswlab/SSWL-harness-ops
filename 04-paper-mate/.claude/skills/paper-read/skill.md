# Paper-Read — 논문 읽기 오케스트레이터

## 트리거 조건

다음 상황에서 이 스킬이 활성화된다:
- 사용자가 `/paper-read` 명령을 입력
- "논문 읽어줘", "논문 번역해줘", "이 논문 좀 봐줘" 등의 자연어 요청
- DOI 주소(예: `10.xxxx/...`)나 논문 제목을 직접 제공
- 논문 파일 경로를 제공

## 실행 프로토콜

### Phase 0: 입력 확보

1. 사용자로부터 논문 소스를 확인한다:
   - DOI 주소가 있는가?
   - 논문 제목이 있는가?
   - 파일을 직접 제공했는가?
2. 작업 경로를 확인한다 (기본값: `_workspace/`)
3. 누락된 정보가 있으면 되물어서 확보한다

### Phase 1: 논문 확보 및 구조 분석

**담당**: paper-fetcher

1. 논문 원문을 확보한다
2. 메타데이터를 추출한다
3. 챕터/섹션 구조를 분석하고 병렬 번역 분할 계획을 수립한다
4. 참고문헌 목록을 추출한다
5. **PDF에서 그림 이미지를 추출한다** (`pdftoppm` 페이지 렌더링 + 필요 시 `pdfimages`). 결과는 `{작업경로}/papers/figures/page-NNN.png`에 저장하고, 그림 목록 표의 `이미지 경로` 칸에 기록한다.

**산출물**: `{작업경로}/00_paper_source.md`, `{작업경로}/papers/figures/*.png`

**진행 보고**: "논문을 확보했습니다. [제목], 총 N개 섹션으로 구성. 번역을 시작합니다."

### Phase 2: 챕터별 병렬 번역 + 그림 분석

**담당**: chapter-translator (×N 병렬) + figure-analyst (병렬)

**챕터 번역** (chapter-translator):
1. Phase 1의 분할 계획에 따라 각 챕터를 **병렬로** 번역한다
2. 각 챕터는 독립적으로 완전 번역된다
3. 각 챕터의 전문용어 목록을 함께 생성한다
4. **수식은 정확한 LaTeX 문법**으로 작성한다 (`$...$`, `\[...\]`, `\begin{equation}` 등). context-harmonizer가 이를 그대로 컴파일한다.

**그림 분석** (figure-analyst):
1. Phase 1에서 추출된 PNG 이미지(`papers/figures/page-NNN.png`)를 Read 도구로 **직접** 열어 시각적으로 분석한다.
2. **캡션·본문 차단 상태에서 먼저 자력 해석**한 뒤, 캡션과 대조하여 일치/불일치를 명시한다.
3. 각 그림의 내용, 맥락, 핵심 인사이트를 한국어로 상세히 설명한다.
4. 그림 간 관계 및 논문의 시각적 서사를 정리한다.
5. 그림이 없는 논문의 경우 이 단계를 건너뛴다.

**병렬 실행 방법**:
- 각 챕터에 대해 별도의 chapter-translator 에이전트를 호출한다
- figure-analyst도 동시에 호출하여 그림 분석을 병렬로 수행한다
- Agent 도구를 사용하여 동시에 여러 챕터 번역 + 그림 분석을 실행한다
- 모든 번역과 그림 분석이 완료될 때까지 대기한다

**산출물**:
- `{작업경로}/01_chapter_translations/ch{N}_{섹션명}.md` (챕터 수만큼)
- `{작업경로}/06_figure_analysis.md` (그림이 있는 경우)

**진행 보고**: "N개 챕터 번역 및 그림 M개 분석 완료. 문맥 통합을 시작합니다."

### Phase 3: 문맥 통합 + LaTeX 컴파일

**담당**: context-harmonizer

1. 모든 챕터별 번역을 수집한다
2. 그림 분석 결과(`06_figure_analysis.md`)를 번역문의 해당 위치에 반영한다
3. 전문용어를 통일한다 (그림 분석에 사용된 용어 포함)
4. 챕터 간 문맥 흐름을 조정한다
5. 하나의 완성된 Markdown 문서(`02_full_translation.md`)를 생성한다
6. **LaTeX 소스(`02_full_translation.tex`)를 작성하고, `tectonic`(또는 `pandoc --pdf-engine=xelatex`)으로 PDF를 컴파일한다.** 한글은 Noto CJK KR로 렌더링한다.
7. 컴파일 로그를 점검하고 수식 오류 발생 시 최대 3회 재시도한다.

**산출물**:
- `{작업경로}/02_full_translation.md` (Markdown)
- `{작업경로}/02_full_translation.tex` (LaTeX 소스)
- `{작업경로}/02_full_translation.pdf` (최종 PDF)

**진행 보고**: "전체 번역과 PDF 컴파일이 완성되었습니다. 참고문헌 분석을 진행합니다."

### Phase 4: 참고문헌 분석

**담당**: reference-analyst

1. 인용 논문을 중요도별로 분류한다 (A/B/C 등급)
2. A등급(필수) 논문에 대해 상세 요약을 작성한다
3. 추천 읽기 순서를 제안한다

**산출물**: `{작업경로}/03_reference_analysis.md`, `{작업경로}/04_must_read_papers.md`

**진행 보고**: "참고문헌 분석 완료. 필수 논문 N편을 선별했습니다."

### Phase 5: Q&A 대기

**담당**: qa-companion

1. 지금까지의 모든 산출물을 참조 가능 상태로 유지한다
2. 사용자의 후속 질문을 받아 답변한다
3. Q&A 내용을 기록한다

**산출물**: `{작업경로}/05_qa_log.md` (누적)

**안내 메시지**:
```
📖 논문 번역 및 분석이 모두 완료되었습니다!

생성된 파일:
- 00_paper_source.md       — 원문 정보 및 메타데이터
- papers/figures/          — PDF에서 추출된 그림 PNG (시각 분석용)
- 01_chapter_translations/ — 챕터별 번역 (번역 노트 포함)
- 02_full_translation.md   — 통합 완성 번역문 (Markdown)
- 02_full_translation.tex  — LaTeX 소스 (수식 포함)
- 02_full_translation.pdf  — 컴파일된 PDF (수식 렌더링 완료)
- 03_reference_analysis.md — 참고문헌 중요도 분류
- 04_must_read_papers.md   — 필수 논문 요약
- 06_figure_analysis.md    — 그림 분석 보고서 (이미지 자체 해석, 그림이 있는 경우)

이제 논문에 대해 궁금한 점을 자유롭게 질문해 주세요!
```

## 에러 핸들링

| Phase | 오류 | 대응 |
|-------|------|------|
| 1 | 논문 확보 실패 | 사용자에게 대안 제시 (다른 소스, 파일 직접 제공) |
| 1 | PDF 이미지 추출 실패 | `pdftoppm` 옵션 조정 후 재시도, 그래도 실패하면 figure-analyst를 폴백 모드(캡션 기반)로 운용 |
| 2 | 일부 챕터 번역 실패 | 실패한 챕터만 재시도, 나머지는 보존 |
| 2 | 그림 분석 실패 | 캡션/본문 기반으로 최선의 분석 수행, 한계 명시 |
| 3 | 용어 통일 충돌 | 가장 널리 사용되는 번역 채택, 기록에 명시 |
| 3 | LaTeX 컴파일 실패 (3회 시도) | `.tex`만 산출하고 PDF 미생성 보고. 컴파일 로그를 사용자에게 전달 |
| 4 | 참고문헌 정보 부족 | 가용 정보 기반으로 최선의 분류 |

## 운영 모드

| 모드 | 설명 | Phase |
|------|------|-------|
| **전체** (기본) | 번역 + LaTeX/PDF + 참고문헌 분석 + Q&A | 0→1→2→3→4→5 |
| **번역만** | 번역 + LaTeX/PDF까지 수행 | 0→1→2→3 |
| **분석만** | 이미 번역된 논문의 참고문헌만 분석 | 4 |
| **Q&A만** | 이미 완료된 결과를 기반으로 질문 응답 | 5 |
| **PDF 재컴파일** | 기존 `02_full_translation.md/.tex`를 PDF로 다시 컴파일 | 3 (5단계만) |

사용자가 모드를 지정하지 않으면 **전체** 모드로 실행한다.
