# 04-Paper-Mate — 논문 읽어주는 하네스

영어 논문이 어려운 사용자를 위해 **학술 논문을 완전히 번역하고, 인용 논문을 분석하며, 후속 질문에 답변**하는 AI 하네스.
사용자가 DOI, 제목, 또는 파일을 제공하면 논문 전체를 챕터별로 병렬 번역한 뒤 하나의 완성된 한국어 문서로 통합한다.

## 에이전트 팀 구성표

| 에이전트 | 역할 | 설명 |
|---|---|---|
| **paper-fetcher** | 논문 확보 | DOI/제목으로 논문 검색 및 다운로드, 또는 사용자 제공 파일 읽기. 논문 구조(챕터/섹션) 및 그림 목록 분석 |
| **chapter-translator** | 챕터 번역 | 개별 챕터/섹션을 한국어로 완전 번역. 병렬 실행으로 속도 확보 |
| **figure-analyst** | 그림 분석 | 논문의 그림(Figure)을 시각적으로 분석하여 내용·맥락·인사이트를 한국어로 상세 설명 |
| **context-harmonizer** | 문맥 통합 | 병렬 번역된 챕터들과 그림 분석을 하나로 합치고, 용어 통일 및 문맥 흐름 조정 |
| **reference-analyst** | 인용 분석 | 참고문헌을 중요도별 분류, 필수 논문 요약 및 정리 |
| **qa-companion** | Q&A 동반자 | 번역 완료된 전체 내용과 그림 분석을 참고하여 사용자의 후속 질문에 답변 |

## 실행 흐름

```
사용자 요청 (DOI / 제목 / 파일 경로)
    │
    ▼
paper-fetcher (논문 확보 & 구조 분석 & 그림 목록 추출)
    │
    ├──► chapter-translator ×N (챕터별 병렬 번역)
    ├──► figure-analyst (그림 분석 — 번역과 병렬 실행)
    │
    ▼
context-harmonizer (통합 & 용어 통일 & 문맥 조정 & 그림 분석 반영)
    │
    ├──► reference-analyst (인용 논문 중요도 분류 & 필수 논문 요약)
    │
    ▼
qa-companion (Q&A 대기 — 사용자의 후속 질문에 답변)
```

## 필수 입력 정책

파이프라인 시작 전 아래 항목을 확보한다. 누락 시 되물어서 확보한다.

| 항목 | 변수 | 용도 |
|---|---|---|
| 논문 소스 | `{소스}` | DOI 주소, 논문 제목, 또는 파일 경로 중 하나 |
| 작업 경로 | `{작업경로}` | 결과물 저장 위치 (기본값: `_workspace/`) |

**논문 소스 확보 우선순위:**
1. DOI 주소 → 웹에서 논문 메타데이터 및 본문 확보 시도
2. 논문 제목 → 웹 검색으로 논문 확보 시도
3. 위 방법으로 확보 불가 시 → 사용자에게 `{작업경로}/papers/` 하위에 PDF/텍스트 파일 직접 제공 요청

## 작업 경로 정책

- **하네스 내 `_workspace/`는 빈 스캐폴드**(디렉토리 구조 템플릿)이다. 실행 결과물을 여기에 저장하지 않는다.
- 이후 모든 `_workspace/` 참조는 사용자가 지정한 `{작업경로}`로 치환된다.
- 사용자가 작업 경로를 별도로 지정하지 않으면 하네스 루트의 `_workspace/`를 사용한다.

## 데이터 전달 규칙

에이전트 간 모든 데이터는 `{작업경로}` 하위 파일로 전달한다.

| 에이전트 | 출력 파일 |
|---|---|
| paper-fetcher | `{작업경로}/00_paper_source.md`, `{작업경로}/papers/` (원본 파일) |
| chapter-translator | `{작업경로}/01_chapter_translations/ch{N}_{섹션명}.md` |
| figure-analyst | `{작업경로}/06_figure_analysis.md` |
| context-harmonizer | `{작업경로}/02_full_translation.md` |
| reference-analyst | `{작업경로}/03_reference_analysis.md`, `{작업경로}/04_must_read_papers.md` |
| qa-companion | `{작업경로}/05_qa_log.md` |

**전달 규칙:**
1. 각 에이전트는 자신의 지정 파일에만 쓴다
2. 다른 에이전트의 출력은 읽기 전용으로 참조한다
3. 모든 중간 산출물은 삭제하지 않고 보존한다

## 번역 원칙

1. **완전 번역**: 논문의 모든 내용을 빠짐없이 번역한다. 요약이나 생략 금지.
2. **원문 충실**: 원문의 의미, 논리 구조, 어조를 있는 그대로 전달한다.
3. **번역체 지양**: "~하는 것이다", "~에 의해" 같은 어색한 번역체를 피하고 자연스러운 한국어로 옮긴다.
4. **전문 용어 처리**: 학술 전문용어는 한국어(영어 원문) 형태로 병기한다. 예: "합성곱 신경망(Convolutional Neural Network)"
5. **수식/그림/표**: 수식은 LaTeX 형태로 보존, 그림/표는 캡션과 설명을 번역한다. 그림이 포함된 논문은 figure-analyst가 시각적으로 분석하여 상세 설명을 제공한다.
6. **병렬 번역 후 통합**: 챕터별로 병렬 번역한 뒤, 전체를 합쳐 용어 통일과 문맥 흐름을 조정한다.

## 사용 언어

- 사용자 대면: 한국어
- 번역 출력: 한국어 (영어 원문 전문용어 병기)
- 파일명/설정: 영어

## 핵심 원칙

1. **완전성**: 논문의 단 한 문장도 빠뜨리지 않는다
2. **정확성**: 학술적 의미를 왜곡하지 않는다
3. **접근성**: 영어에 익숙하지 않은 사용자가 이해할 수 있도록 자연스럽게 번역한다
4. **참고문헌 가이드**: 인용 논문의 중요도를 분류하여 후속 학습 로드맵을 제공한다
5. **지속 지원**: 번역 완료 후에도 내용 기반 Q&A를 지원한다
