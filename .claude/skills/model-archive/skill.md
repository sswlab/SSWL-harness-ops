# Model-Archive — 모델 아카이빙/활용 오케스트레이터

## 개요

연구실에서 개발된 모델들을 표준화된 형식으로 **등록·관리**하고, 누구나 자연어로 **호출하여 결과를 얻을 수 있게** 한다. 개발에 참여하지 않은 연구자도 동일하게 사용 가능하다.

## 에이전트 구성표

| 에이전트 | 역할 | 호출 Phase |
|---|---|---|
| **skill-archivist** | 모델 분석, 카드 생성, 등록/업데이트 | Phase 1~3 |
| **query-planner** | 등록 모델 활용 시 실행 계획 수립 | Phase A1 |
| **data-fetcher** | 모델 실행에 필요한 데이터 수집 | Phase A3 내부 |
| **model-runner** | 등록된 모델 실행 | Phase A3 내부 |
| **task-executor** | 계획에 따른 작업 실행 | Phase A3 |
| **result-reporter** | 결과 보고 | Phase A4 |

## 워크플로우 A: 모델 등록

```
[개발자: "모델 등록해줘"]
    │
    ▼
╔═══════════════════════╗
║   Phase 1: 분석       ║
║   skill-archivist     ║
║   (코드 분석, 의존성)   ║
╚════════╤══════════════╝
         │
         ▼
╔═══════════════════════╗
║   Phase 2: 표준화     ║
║   skill-archivist     ║
║   (모델 카드 + 래퍼)   ║
╚════════╤══════════════╝
         │ 모델 카드 초안
         ▼
╔═══════════════════════╗
║   Phase 3: 검증       ║
║   skill-archivist     ║
║   (테스트 실행)        ║
╚════════╤══════════════╝
         │ 등록 완료
         ▼
[registry_index.json 갱신]
```

## 워크플로우 B: 등록 모델 활용

등록된 모델을 사용하는 흐름은 **research-task와 동일**하되, query-planner가 model_registry를 참조한다:

```
[연구자: "코로나홀 탐지 모델로 어제 데이터 분석해줘"]
    │
    ▼
╔═════════════════════════════╗
║   Phase A1: 계획 수립       ║
║   query-planner             ║
║   (model_registry 참조)     ║
╚════════╤════════════════════╝
         ▼
╔═════════════════════════════╗
║   Phase A2: 사용자 승인     ║
╚════════╤════════════════════╝
         ▼
╔═════════════════════════════╗
║   Phase A3: 실행            ║
║   task-executor             ║
║   ├ data-fetcher (필요 시)   ║
║   └ model-runner (모델 카드  ║
║     참조하여 실행)           ║
╚════════╤════════════════════╝
         ▼
╔═════════════════════════════╗
║   Phase A4: 결과 보고       ║
║   result-reporter           ║
╚═════════════════════════════╝
```

## 워크플로우 C: 모델 목록 조회

```
[연구자: "우리 연구실에 어떤 모델들이 있어?"]
    │
    ▼
[skill-archivist]
    │ registry_index.json 읽기
    ▼
[모델 목록 표/카드 형식으로 제시]
```

## 워크플로우 D: 모델 업데이트

```
[개발자: "코로나홀 탐지 모델 v2로 업데이트해줘"]
    │
    ▼
[skill-archivist]
    ├── 기존 버전 아카이브 (versions/v1.0/)
    ├── 새 코드 분석
    ├── 모델 카드 업데이트
    ├── 테스트 실행
    └── registry_index.json 갱신
```

## 모델 레지스트리 구조

```
_workspace/model_registry/
├── registry_index.json
├── {model_name}/
│   ├── model_card.md
│   ├── run.sh
│   ├── src/
│   └── versions/
│       └── v{N}/
```

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 모델 경로 미존재 | 경로 확인 요청 |
| 의존성 충돌 | 가상환경 분리 제안 |
| 테스트 실행 실패 | 원인 분석 후 수정 안내 |
| 등록 모델 실행 실패 | model_card.md의 제한사항 확인 안내 |
| 중복 모델명 | 버전 업데이트인지 확인 |
